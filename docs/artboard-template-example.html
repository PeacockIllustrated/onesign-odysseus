AI-friendly artboard annotation

This is the canonical format for shopfront artboards so the AI extractor can reliably turn them into production jobs. Use this layout for fascia signs, window vinyls, door vinyls, and projection signs. The AI will read each component’s card, extract its sub-items, and pre-populate the job. Human review still required before anything is committed.

Example shopfront — BRIGHT & CO

COMPONENT 1

COMPONENT 2

Rule of thumb: one physical assembly = one component. The fascia is one component even though it has both acrylic and vinyl on it — because it’s one panel that gets installed as one unit. The door is one component because the vinyls get applied to the same door. Components split when they install separately. Each component lives on its own “card” on the artboard, as shown in pages 2–3.

Rules that make extraction reliable

The AI reads your artboard left-to-right, top-to-bottom, card by card. Keeping the structure below means it maps cleanly into a sub-item with high confidence. Deviations don’t break extraction — they just drop confidence and force manual review of more fields.

Do

  • One physical assembly per component card. Fascia = one card, projection sign = one card.
  • Label every sub-item with a single capital letter (A, B, C…) at the callout.
  • Always use the key: value format with these exact labels: Material:, Method:, Finish:, W × H:, Returns:, Qty:, Dept:.
  • Every dimension includes the unit: 1500 × 280 mm. Never bare numbers.
  • Name the sub-item in a short readable phrase — “BRIGHT & CO letters”, not just “A”.
  • State the target department explicitly. The model should not guess this.
  • Keep annotation text typeset. Avoid handwriting.
  • Put the component name + type in a visible header on each card (“Component 1 · Fascia sign”).

Don’t

  • Don’t mix two components on one card. If it installs separately, split it.
  • Don’t use numbers (1, 2, 3) for sub-items. Numbers are for components.
  • Don’t write prose: “big letters in rose gold acrylic about 1.5m across stuck on the front” forces the model to guess at parsing.
  • Don’t omit units. 1500 x 280 without “mm” is ambiguous.
  • Don’t skip the material when it’s “obvious from the image” — the model can’t tell acrylic from composite by looking.
  • Don’t use ambiguous arrows that cross another callout’s line.
  • Don’t rely on colour alone to differentiate sub-items. Colour doesn’t survive greyscale prints or screenshots.
  • Don’t cross-reference between components (“same material as Component 1”). Restate the spec on each card.

Side-by-side — reading the same element two ways

Extractable A. BRIGHT & CO letters Material: 5mm rose-gold mirrored acrylic Method: stuck to face with VHB tape Finish: rose gold mirror W × H: 1500 × 280 mm Qty: 1 Dept: CNC / acrylic finishing
Will produce a wrong job Letters — rose gold, ~1.5m, stuck on (the rest as usual)

Two different annotations describing the same physical thing. The first extracts as a clean sub-item. The second produces a sub-item with “inferred” markers on material, dimensions, method, finish, and department — which is exactly the situation the AI feature is not meant to replace human review for. If Davey writes the first format consistently, extraction becomes reliable. If the second format creeps in, the tool stops saving time because the reviewer has to fill in everything that was left vague.

One-page reference

Pin this up in the studio. The shape below is all the model needs to produce a clean sub-item with no guesses.

Component N · [Client name] · [Type: fascia | window | door | projection | lightbox | …] A. [Short readable name of this sub-item] Material: [exact spec, including thickness / brand / colour where relevant] Method: [how it is applied, fixed, or assembled] Finish: [surface finish / colour / texture] W × H: [width] × [height] mm Returns: [depth] mm ← only when relevant (e.g. built-up letters) Qty: [integer] Dept: [one of: CNC / acrylic finishing · Vinyl plotter · Print / finishing · Fabrication · Electrical · Install] B. [Next sub-item in the same component, same shape] …

Component type shorthand

These are the canonical component-type words the model recognises. Using other words is fine but reduces confidence on the “component type” field.

fascia — horizontal signboard above shopfront
window — vinyl or film applied to glazing
door — vinyls / fittings on the entrance door
projection — wall-mounted perpendicular sign
lightbox — illuminated face-lit or edge-lit unit
built_up_letters — standalone 3D letters
flat_cut_letters — flat acrylic / composite letters
totem — free-standing external sign
wayfinding — internal directional signage

Department shorthand

Use these exact department names — they map 1:1 to stages in the production pipeline. If the department isn’t in this list, the sub-item will come back with inferred on Dept and Davey (or whoever reviews the draft) has to pick one manually.

• CNC / acrylic finishing
• Vinyl plotter
• Print / finishing
• Fabrication
• Electrical
• Install
